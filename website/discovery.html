Discovery Engine

Automated overnight parameter search using walk-forward validation, genetic programming, and alternative data integration. The system discovers and validates its own signal improvements weekly.

§ 01 — GENETIC INDICATOR DISCOVERY

Expression Tree Evolution

The genetic engine evolves indicator expressions as binary expression trees. Each tree combines primitives from a fixed library — price transforms, rolling statistics, arithmetic operators — into novel indicator formulas that no human explicitly designed.

Each generation applies tournament selection, subtree crossover, and point mutation. Trees that produce higher Information Coefficient scores on walk-forward out-of-sample folds survive. After 20 generations, any tree with IC > 0.05 graduates to the discovered_indicators table for human review.

§ 02 — WALK-FORWARD IC SCORING

Out-of-Sample Validation Protocol

Information Coefficient (IC) measures the Spearman rank correlation between a signal's predicted direction and the actual realized return over the next holding period. IC = 0 means the signal is noise; IC = 1 means perfect prediction.

Walk-forward validation prevents look-ahead bias. A 6-year data window is divided into 5 folds of 4 months each. The model trains on all prior folds, then predicts on the next unseen fold. IC is computed on the out-of-sample prediction only — in-sample fit is irrelevant.

IC > 0.05 is the graduation threshold. This is a relatively modest bar — it means roughly 52.5% directional accuracy over the test window. The requirement is that this threshold holds consistently across all 5 folds, not just the best one.

IC Formula IC = SpearmanCorr(signal_rank, return_rank)
fold IC = mean IC across 4-month window
Spearman rank is used over Pearson to reduce outlier sensitivity in fat-tailed return distributions

Form 4 is filed within 2 business days of any insider trade. The bot polls the EDGAR XBRL API continuously, parsing XML for transaction type (P/S), shares transacted, and per-share price.

Cluster detection fires when multiple insiders at the same company buy within a rolling 30-day window. Transactions above $1M trigger auto-signal generation. Tiered scoring by officer rank: CEO/CFO weighted highest.

FINRA CNMS · Short Volume
Daily Short Interest Ratio
FINRA publishes daily consolidated short volume data (CNMSshvol) per symbol. The ratio of short volume to total volume is computed for each symbol in the watchlist.

A ratio ≥ 65% vetoes any long entry as a hard GATE-07 block. When a high-short-ratio stock shows a price uptick relative to prior day, the system flags a potential short squeeze and posts to the Slack alerts channel.

§ 04 — WEEKLY RUN SCHEDULE

Overnight Discovery Pipeline

Two discovery loops run on independent schedules. v2 parameter search runs every Friday afternoon before market close data is finalized. The genetic programming loop runs Saturday night across the full 6-symbol universe.

[FRIDAY 4:30PM] discovery_loop() fires — v2 parameter search begins
[ENGINE] 1,215+ param combos tested · 5 strategy families · top-100 S&P
[VALIDATION] Walk-forward Sharpe per regime (bull · bear · high-vol)
[RESULTS] bull_sharpe · bear_sharpe · high_vol_sharpe → JSONB
[STORAGE] discovery_results table · status=pending_approval
[APPROVAL] human review via Streamlit dashboard · approve / reject per symbol
[SATURDAY 11PM] indicator_discovery_loop() fires — GP engine begins
[GENETIC] 50 trees × 20 generations × 6 symbols · ~18,000 evaluations
[IC GATE] IC > 0.05 required across all walk-forward folds to graduate
[LIBRARY] discovered_indicators table updated with graduated expressions
[SLACK] Summary posted to #performance channel with top IC scores
